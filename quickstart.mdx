---
title: "Quickstart"
description: "Get Airport Manager running locally in under 5 minutes."
---

<Steps>
  <Step title="Prerequisites">
    Ensure you have the following installed:

    - **Python 3.10+** — [python.org](https://www.python.org/downloads/)
    - **pip** — comes bundled with Python
    - **git** — [git-scm.com](https://git-scm.com/)
  </Step>

  <Step title="Clone the repository">
    ```bash
    git clone https://github.com/egarcia-gf/airport-manager.git
    cd airport-manager
    ```
  </Step>

  <Step title="Create and activate a virtual environment">
    <Tabs>
      <Tab title="macOS / Linux">
        ```bash
        python -m venv venv
        source venv/bin/activate
        ```
      </Tab>
      <Tab title="Windows">
        ```bash
        python -m venv venv
        venv\Scripts\activate
        ```
      </Tab>
    </Tabs>
  </Step>

  <Step title="Install dependencies">
    ```bash
    pip install -r requirements.txt
    pip install requests python-dotenv
    ```

    `requirements.txt` installs the core Django packages:

    | Package | Version | Purpose |
    |---------|---------|---------|
    | Django | 6.0.3 | Web framework |
    | asgiref | 3.11.1 | ASGI support for Django |
    | sqlparse | 0.5.5 | SQL formatting for Django ORM |

    <Note>
      `requests` (used in `aeropuertos/views.py` to call the AirportGap API) and `python-dotenv` (used in `settings.py` to load `.env`) are not listed in `requirements.txt`. Install them separately with the command above.
    </Note>
  </Step>

  <Step title="Configure environment variables">
    Copy the example env file and add your AirportGap API token:

    ```bash
    cp .env.example .env
    ```

    Edit `.env`:

    ```bash .env
    AIRPORT_GAP_TOKEN=your_real_token_here
    ```

    <Note>
      The `.env.example` file ships with a placeholder value (`aqui_va_un_token`). You must replace this with a real token.
    </Note>

    To get your token, register a free account at [airportgap.com](https://airportgap.com) and copy your API token from your account settings.

    <Warning>
      The `SECRET_KEY` in `airport_manager/settings.py` is hardcoded and insecure. For any deployment beyond local development, generate a new secret key:

      ```bash
      python -c "from django.core.management.utils import get_random_secret_key; print(get_random_secret_key())"
      ```

      Then set it as an environment variable rather than hardcoding it.
    </Warning>
  </Step>

  <Step title="Run database migrations">
    ```bash
    python manage.py migrate
    ```

    This creates the local SQLite database (`db.sqlite3`) used by Django for sessions and the admin panel.
  </Step>

  <Step title="Start the development server">
    ```bash
    python manage.py runserver
    ```

    Expected output:

    ```
    Watching for file changes with StatReloader
    Performing system checks...

    System check identified no issues (0 silenced).
    Django version 6.0.3, using settings 'airport_manager.settings'
    Starting development server at http://127.0.0.1:8000/
    Quit the server with CONTROL-C.
    ```
  </Step>

  <Step title="Open the app">
    Open your browser at [http://127.0.0.1:8000](http://127.0.0.1:8000).

    <Tip>
      All available routes:

      | URL | Feature |
      |-----|---------|
      | `http://127.0.0.1:8000/` | Airport search |
      | `http://127.0.0.1:8000/favoritos/` | My favorites |
      | `http://127.0.0.1:8000/distancia/` | Distance calculator |
      | `http://127.0.0.1:8000/comparar/` | Airport comparison |
    </Tip>
  </Step>
</Steps>
